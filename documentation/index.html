EMBER — Documentation

Luxury Restaurant HTML Template · Version 1.0

Thank you for purchasing EMBER! This documentation will guide you through customizing the template for your restaurant. If you have any questions beyond what's covered here, please contact me via my ThemeForest profile.

⚡ Quick Start: This is a static HTML template. You can edit it with any code editor (VS Code, Sublime Text, Notepad++). No build tools or npm required.

🚀 Getting Started

1. Open the Files

Open the entire folder in your code editor. I recommend VS Code (free).

2. Preview Locally

Double-click index.html to open it in your browser. You'll see the template with placeholder content.

3. Start Editing

Open index.html and start replacing the placeholder text with your restaurant's information. The most common edits are covered below.

🖼️ Replacing Images

All images are stored in the /images/ folder. To replace an image:

  1. Add your new image to the /images/ folder
  2. Find the <img> tag in index.html referencing the old image
  3. Change the src attribute to your new filename

🎨 Changing Colors

All colors are defined as CSS variables at the top of styles.css. Change them once, and the entire site updates.

Open styles.css and find this section (lines 5-14):

:root {
  --ember: #ff4500;      /* Primary accent — buttons, highlights */
  --gold: #c29723;       /* Secondary accent — borders, prices */
  --deep: #0a0602;       /* Darkest background */
  --ash: #130e06;        /* Slightly lighter background */
  --cream: #f5edd8;      /* Main text color */
  --flame: #ff6b35;      /* Hover states */
  --nav-h: 80px;
  --pad-x: 60px;
}

Example: Changing from Orange to Blue Theme

:root {
  --ember: #0066cc;      /* Changed from #ff4500 */
  --gold: #c29723;       /* Keep gold accents or change */
  --flame: #3388ff;      /* Changed from #ff6b35 */
  /* ... rest unchanged ... */
}

Save the file, refresh your browser, and all orange elements become blue.

🍽️ Updating Menu Items

Menu items are organized in tabs (Starters, Mains, Desserts, Wine). Each item follows this structure:

<div class="mc">
    <div class="mc-photo">
        <div class="img-wrap">
            <img src="images/photo-7.webp" alt="Wagyu Tartare" loading="lazy">
        </div>
    </div>
    <div class="mc-ov"></div>
    <span class="mc-num" aria-hidden="true">01</span>
    <div class="mc-body">
        <div class="mc-name">Wagyu Tartare</div>
        <div class="mc-desc">
            Hand-cut A5 Wagyu, smoked egg yolk, black truffle...
        </div>
        <div class="mc-foot">
            <span class="mc-price">EGP 1,250</span>
            <span class="mc-pill">Chef's Pick</span>
        </div>
    </div>
</div>

To add a new menu item:

  1. Copy an entire <div class="mc"> block
  2. Paste it before the closing </div> of the menu panel
  3. Update the image, name, description, price, and pill text
  4. Update the mc-num to the next number

To remove an item, simply delete its entire <div class="mc"> block.

📅 Reservation Form Setup

The reservation form is currently frontend-only with validation. To make it actually send emails, you have several options:

Option 1: Formspree (Easiest, Free Tier)

  1. Go to formspree.io and create a free account
  2. Create a new form and get your form ID (looks like xaygnkyo)
  3. In index.html, find the reservation form (search for id="resForm")
  4. Add these attributes to the <form> tag:
<form action="https://formspree.io/f/YOUR_FORM_ID" method="POST">

Make sure all inputs have name attributes. Add them if missing:

<input type="text" id="fname" name="first_name" ...>
<input type="email" id="remail" name="email" ...>

Option 2: Google Sheets (Free, Unlimited)

Use a service like Sheetmonkey to send form submissions directly to Google Sheets.

Option 3: PHP Backend (If You Have Server Access)

Create a send-reservation.php file and point the form action to it.

📧 Current Behavior: Without a backend, the form shows a "Confirmed" message but doesn't actually send data anywhere. This is fine for demo purposes, but set up Formspree before going live.

🗺️ Google Maps

The map is embedded via an iframe. To update it with your restaurant's location:

  1. Go to Google Maps and search for your address
  2. Click "Share" → "Embed a map"
  3. Copy the iframe code
  4. In index.html, search for maps/embed?pb=
  5. Replace the entire src attribute value with your copied code's src
<iframe 
    src="YOUR_NEW_EMBED_URL"
    allowfullscreen="" 
    loading="lazy">
</iframe>

🔍 SEO Settings

Update the meta tags in the <head> section of index.html:

<meta name="description" content="Your restaurant description here...">
<meta property="og:title" content="YOUR RESTAURANT NAME — Fine Dining">
<meta property="og:description" content="Your description...">

🔖 Changing the Favicon

The favicon is currently an inline SVG emoji. To use your own logo:

  1. Create a favicon at favicon.io
  2. Save the generated files to your /images/ folder
  3. Replace the favicon link in index.html (around line 15):
<link rel="icon" type="image/x-icon" href="images/favicon.ico">

🔧 Troubleshooting

Images aren't loading

  • Check that image filenames match exactly (case-sensitive)
  • Verify images are in the /images/ folder
  • Try using .jpg instead of .webp if your server doesn't support WebP

Custom cursor isn't working

  • Custom cursor only appears on desktop (screen width > 768px)
  • If you're on a touchscreen device, it's disabled by design

Reservation form shows errors but fields are filled

  • Make sure you selected a time and number of guests (not just typed)
  • Email must be in format: name@domain.com

Colors aren't changing after editing CSS variables

  • Hard refresh your browser (Ctrl+Shift+R or Cmd+Shift+R)
  • Check for typos in the color codes (must start with #)
